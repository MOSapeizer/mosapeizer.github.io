最近把 terminal 換去 Ghostty。

很爽，推推推

我一直都在用 Warp (我知道很貴也很爛) 它的 hotkeys 設計真的不行，導致我一直用 Cmd+I 在 agent mode 之間切換

我只是要輸入一些常見的 commands 也會誤切到 agent mode 真的超煩

原本想切回 WezTerm，但調查後覺得既然要改，就乾脆改成 S tier 的。

測試了兩天，使用體驗還不錯，也可能是我適應了新的 mac window 管理方法所以整體用起來變得很順

現在 tiling window 流程就是 raycast 快速的找到 app, 然後把常用的 app 綁上 ctrl 1-9
, macos native window move left or right

開發時也只需要 terminal 就在 ghostty 裡面 split windows 只需要 claude code + lazyvim 很多事情就能解決了

解決完這些，還有一件事是最近很困擾的

那就是我發現自己驗證 AI 產出速度很慢

為了找到比較結構化的方法我把手邊已經派出去的工作都收回來自己做順便測試幾個做法：

大概是這三種

1. one-shot solution 就乾脆不多驗，只要正流程有做到就好。
2. 偏維護的專案就只看 testcases、code coverage，最多再驗一下 benchmarks。
3. 陌生功能的開發就加 AI agents，或多利用 Claude skills，loop 幾次濃縮成果後我再檢查。

1 的話我都用 claude code，雖然 codex cli 不差，但結果總是 opus 寫的又快又對

但 3 的效果比想像中差，不是預期的 1 → 1.5 → 1.5，
更多時候是 1 → 0.85 → 1.1 這種結果。
